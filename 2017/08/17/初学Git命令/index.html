初学Git命令

github远程提交基本说明
git config --global user.email "github邮箱"
git config --global user.name "github名字"
首先全局配置下自己的邮箱和名字

git init
初始化一个仓库，但是一般常用的都是克隆现有的仓库

git clone 远程仓库地址(例如git@github.com:geniustesda/python-exercise.git)
克隆目标的远程仓库内容

git status
查看当前仓库的状态
git status -s
简化查看当前仓库的状态

touch Newfile
新建一个文件
git rm Newfile
删除一个文件
git mv Newfile ../
将目标文件移动到上级目录

git add Newfile
将目标文件加入到本地工作目录缓存区

git add .
将当前目录所有文件加入到缓存区

git commit -m "提交的注释"
将缓存区文件提交到本地仓库，并加入注释

git push -u origin master
将缓存区文件提交到远程仓库

master是你的分支，git origin是你的远程仓库
第一次git push -u origin master后，就可以用直接用git push指令了

SSH:

ssh-keygen -t rsa -C "你的github的邮箱"
生成本地SSH文件

(因为之前有一个original的，我们这里方便测试先删了它)
git remote rm origin
输入
git remote
发现没东西了，可以添加新的远程仓库了。
git remote add origin 远程仓库地址(例如git@github.com:geniustesda/python_crawler.git)

Git fetch和git pull的区别

Git中从远程的分支获取最新的版本到本地有这样2个命令：
1. git fetch：相当于是从远程获取最新版本到本地，不会自动merge

Git fetch origin master
git log -p master..origin/master
git merge origin/master

以上命令的含义：
首先从远程的origin的master主分支下载最新的版本到origin/master分支上
然后比较本地的master分支和origin/master分支的差别
最后进行合并
上述过程其实可以用以下更清晰的方式来进行：

git fetch origin master:tmp
git diff tmp
git merge tmp

从远程获取最新的版本到本地的test分支上
之后再进行比较合并

2. git pull：相当于是从远程获取最新版本并merge到本地

git pull origin master
上述命令其实相当于git fetch 和 git merge
在实际使用中，git fetch更安全一些
因为在merge前，我们可以查看更新情况，然后再决定是否合并
